# Data Splits

Pre-computed train/val/test splits in JSON format, generated by `split_data.py` or `split_data_reproducible.py` at the repository root.

## Split Design

Splits are defined at the **organoid level** (not by individual image/timepoint) to prevent data leakage. The same organoid across all days stays in one partition. Final ratios: 72% train / 8% val / 20% test.

Inclusion criteria:

- BA1 and BA2 batches only (high-quality data)
- 4/5 survey vote consensus for the Dy30 label
- Complete metabolite data for all required assays
- Valid processed image and mask paths

## Files

| File | Description |
|------|-------------|
| `both_train_base.json` | Training set, base filtering (no image-type exclusion) |
| `both_val_base.json` | Validation set, base filtering |
| `both_test_base.json` | Test set, base filtering |
| `splits.csv` | Simplified view: `organoid_id`, `split`, `label` columns only |

## Regeneration

```bash
python split_data.py
python split_data_reproducible.py --seed 42
```

See the [root README](../README.md#data-splits) for details on the split logic.
