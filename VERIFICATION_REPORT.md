## Section 3: Cross-Cutting Issues

### Issue A — Organoid Count (265 vs 220) ⚠

The paper states 265 organoids. The current repo uses 220. The discrepancy arises from filtering differences:

| Filter applied | Count |
|---------------|-------|
| Total organoids in all_data.json | 503 |
| BA1 + BA2 batches only | 309 |
| BA1 + BA2 with Day 30 record | 283 |
| BA1 + BA2 with survey at Day 30 | 274 |
| BA1 + BA2 with ≥4/5 vote consensus at Day 30 | **220** (current repo) |

The 54 organoids lost (274→220) are 3-2 vote splits that fail the ≥4 consensus threshold. The paper's 265 likely comes from an older dataset version or a less strict consensus criterion. Feedback from reviewers also suggests 248 as the "correct" number (expert-labeled).

**Impact:** Table 3 aggregate statistics (Avg Accuracy, Avg Bal Acc, Avg Recall) differ from the paper because they are computed over a different test set. The headline Dy30 LightGBM balanced accuracy (0.9444) still matches exactly.

🔧 **Action:** Team alignment needed on the correct organoid filter criterion. Once resolved, Table 3 aggregates should be recomputed.

### Issue B — Malate Std Dev Discrepancy ⚠

Malate std dev in the paper is 0.772. Our computation yields a value off by 0.036. Likely cause: dilution-correction data is pending from Promega (STATUS.md), and/or the exact Malate availability cutoff (Day 10 vs Day 13) differs between runs.

🔧 **Action:** Recompute after dilution-corrected metabolite data arrives.

### Issue C — Table 2 Metrics Not Stored in Current Outputs ✗

The `metrics_test.json` files saved by the image classifier save only overall accuracy and F1. They do not save TNR (true negative rate), per-class recall, or balanced accuracy. This means Table 2 cannot be regenerated from any existing output file. Scripts for all three backbones exist (`train_model_accuracy.py`, `train_model_accuracy_tony_dinov2.py`) but must be extended to save these metrics.

🔧 **Action:** Add TNR and balanced accuracy to the metrics saved in image classification training scripts.

### Issue D — Fig 7 and Fig 8 Carry-Over Images ⚠

Both `perday_vs_timeseries.png` and `three_model.png` were carried over from the original student analysis and are not generated by the current codebase. The infrastructure exists:
- Time-series: CNN-LSTM scripts in `analysis/images/cnn_lstm/`
- Combined: `analysis/multimodal/train_multimodal.py` + `analysis/three_model_plot.py`

But neither has been re-run to completion, so these figures cannot currently be regenerated. The images in `paper/Images/` match the paper and should be treated as correct until re-runs are complete.
