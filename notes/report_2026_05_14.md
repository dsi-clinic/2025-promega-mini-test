## Summary

This week's goal was to rerun both the per-day image classifier (EfficientNet-B0) and the

---

## 1. Image Path Correction

**The problem:** The per-day EfficientNet was loading images from the `overlay_path` /
`img_path` keys in `all_data.json`, which point to `resized_512x384/` — images that are
resized without preserving aspect ratio (organoids are stretched/distorted).

**Correct paths (confirmed with Amanda):**

| Model | Image key | Directory | Resolution |
|-------|-----------|-----------|------------|
| Per-day EfficientNet | `clipped_meanfill.cm_source_image_abs` | `resized_575_square/` | 575 × 575, AR-conserved |
| CNN-LSTM temporal | `lstm_ready/` images (existing in `organoid_dataset.py`) | `lstm_ready/` | already AR-conserved |

The CNN-LSTM was already using the correct paths via `OrganoidTimeSeriesDataset`; only the
per-day classifier needed fixing.

---

## 2. Changes to `pipeline/data_loader.py`

Two additions to support the AR-conserved image paths:

**a) New entries in `IMAGE_MODE_TO_PATH_KEY`:**

```python
IMAGE_MODE_TO_PATH_KEY = {
    "img":            "img_path",
    "mask":           "mask_path",
    "overlay":        "overlay_path",
    "cm_source_image": ("clipped_meanfill", "cm_source_image_abs"),  # ← new
    "cm_source_mask":  ("clipped_meanfill", "cm_source_mask_abs"),   # ← new
}
```

The nested tuple format handles the two-level JSON key
(`record["images"]["clipped_meanfill"]["cm_source_image_abs"]`).

**b) Nested key lookup in `get_image_paths()`:**

```python
if isinstance(path_key, tuple):
    parent, key = path_key
    path = (imgs.get(parent) or {}).get(key)
else:
    path = imgs.get(path_key)
```

**c) Default mode changed** from `"overlay"` to `"cm_source_image"` in both
`get_image_paths()` and the `--input-mode` argument of `perday_image_study.py`.

Dataset: IDOR cohort, 266 organoids with complete 11-day series  
Metric: balanced accuracy (derived from test_recall and FP/FN counts in results JSON)

| Series through Day | Bal Acc (test) | Test F1 | AUC |
|--------------------|---------------|---------|-----|
| 3–8  | ~0.50 | 0.312 | — |

Best model (3–30): test accuracy 88.9%, 2 false positives (BA2 96_2 A3, BA2 96_2 H3),

(3–30) gives AUC 0.918 and balanced accuracy 0.855, outperforming the best single-day
classifier (Dy30: 0.833).

### 5c. Combined Comparison

Both models show the same biological signal: **organoid fate becomes visually distinguishable
around Day 20–21.** Early-day images carry insufficient visual information to distinguish
Acceptable from Not Acceptable organoids.

The time-series model's advantage over the per-day classifier appears primarily in the
later days (24–30), where having the full developmental trajectory provides a small but
consistent improvement. See `$ANALYSIS_OUTPUT_DIR/figures/image_model_comparison.png`.
