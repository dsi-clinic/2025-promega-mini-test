## Issues Fixed Since Last Report

### 1. CNN-LSTM was using a random runtime split
`make_idor_series_splits` was calling `split_organoids(seed=42)` to generate a fresh random
split at runtime, meaning the CNN-LSTM test set had no overlap guarantee with the per-day
model's test set.

**Fix:** Pass `splits=Splits.canonical()` directly to `OrganoidDataset`. The dataset
constructor intersects the IDOR filter with the canonical split automatically. Both models
now evaluate on the same organoids.

```python
# analysis/images/cnn_lstm/organoid_dataset.py
dataset = OrganoidDataset(
    all_data_path,
    filters=filters_for_mode("series_idor"),
    splits=Splits.canonical(),   # ← was: split_organoids(seed=42)
)
```

### 2. CNN-LSTM was using the wrong image key
`get_clipped_meanfill_image_path` returned `cm_image_abs` (mean-filled images in
`mean_fill_clip/`). Per Amanda's instructions, the correct key is `cm_source_image_abs`
(raw AR-conserved images in `resized_575_square/`).

**Fix:** One-line change in `pipeline/data_loader.py`:

```python
# Before
return (...).get("cm_image_abs")
# After
return (...).get("cm_source_image_abs")
```

### 3. Day 28 missing from temporal ablation
`DAY_RANGES` in `train_temporal_ablation_attn.py` was `[8, 10, 13, 15, 17, 20.5, 24, 30]`
— Day 28 was simply absent. Script glitch, not missing data.

**Fix:** `DAY_RANGES = [8, 10, 13, 15, 17, 20.5, 24, 28, 30]`

### Analysis

**Both models at chance through Day 17.** The per-day classifier dips below chance at
Day 10 (0.42) while the time series stays flat at 0.50 — neither model can distinguish
organoid quality from early images alone.

**Sharp inflection at Day 20.5.** Both models jump simultaneously, confirming the
biological signal: organoid fate becomes visually identifiable around Day 20–21,
regardless of the modeling approach.

**Per-day leads through Days 20.5–24.** The shaded region shows the per-day model
outperforming the time series by +0.19 at Day 20.5 and +0.07 at Day 24. This is
counterintuitive — the time series model sees the full developmental history and should
do better. The gap likely reflects the AUC drop (0.918 → 0.791) caused by the smaller
canonical training set and/or the switch from mean-filled to raw images.

**Models converge at Day 28 (both 0.83), then diverge at Day 30.** Per-day improves
to 0.83 while time series drops to 0.79. The time series model peaks at Day 28, not
Day 30 — adding the final two days of data hurts it, suggesting possible overfitting
on the smaller training set.

**Key question for Nick:** The per-day model currently outperforms the time series
model after the corrections. The `cm_image_abs` vs `cm_source_image_abs` question is
now urgent — the previous CNN-LSTM run with mean-filled images gave AUC 0.918 vs 0.791
now. Need to isolate whether the image type or training set size is driving the
degradation before drawing conclusions from this comparison.
