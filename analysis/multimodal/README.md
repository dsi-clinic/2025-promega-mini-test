# Multimodal Organoid Quality Classifier

Combines microscopy image features (CNN) and metabolite assay data to predict organoid quality (Acceptable / Not Acceptable).

## Code Structure

| File | Responsibility |
|------|---------------|
| `multimodal_model.py` | All model and data components: dataset, network modules, training/eval loops |
| `train_multimodal.py` | Training orchestration: shared pretraining, per-day training, argparse, result saving |
| `run_multimodal.s` | SLURM job array script for hyperparameter sweeps |

## Model Architecture

Both modalities are projected to the same `proj_dim` before fusion so neither branch dominates (image backbones produce 1024–2048-d while raw metabolite vectors are only 4–10 values).

```
Image (RGB, 512×384)
    └─► Image Backbone (ResNet50 / EfficientNet-B0 / ViT-B/16)
            └─► backbone features (e.g. 2048-d for ResNet50)
                    └─► img_proj  (Linear→ReLU→Dropout)
                            └─► proj_dim-d  (default 128)

Metabolite concentrations (4–10 values per day, see below)
    └─► MetaboliteBranch MLP  (Linear→ReLU→Dropout→Linear→ReLU)
            └─► proj_dim-d  (default 128)

Fusion (concat or gated)
    └─► Classification Head  (Linear→ReLU→Dropout→Linear)
            └─► binary output: Acceptable (1) / Not Acceptable (0)
```

### Metabolite features

**Base mode** (default):
- Days ≤10: 4 features — Glucose, Glutamate, Lactate, Pyruvate
- Days >10: 5 features — above + Malate

**With growth features** (`--use-growth-features`):  
Appends the delta (concentration change from the previous timepoint) for each base feature. NaN on the first timepoint per organoid is filled with 0.
- Days ≤10: 8 features — 4 base + 4 growth
- Days >10: 10 features — 5 base + 5 growth

`BCAAGlo` and `*_initial_concentration` fields are never used.

### Fusion strategies
| Strategy | How it works |
|----------|-------------|
| `concat` | Image and metabolite proj_dim vectors are concatenated → 2×proj_dim into the head |
| `gated`  | Metabolite vector produces a sigmoid gate that element-wise scales the image vector → proj_dim into the head |

### Backbone options
| Key | Model |
|-----|-------|
| `resnet` | ResNet-50 (pretrained ImageNet) |
| `efficientnet` | EfficientNet-B0 (pretrained ImageNet) |
| `vit` | ViT-B/16 (pretrained ImageNet, img_size adapted to 384×512) |

## Training Process

Training happens in two stages:

### Stage 1 — Shared backbone pretraining (all days combined)
All training samples across every timepoint are pooled together to pretrain the full model end-to-end. This gives the backbone a domain-adapted starting point before day-specific fine-tuning.

### Stage 2 — Per-day classifier heads (backbone frozen)
The backbone is frozen and a separate classification head is trained for each day (Dy03, Dy06, …, Dy30). This captures day-specific signal in the metabolite and image distributions without overfitting the shared feature extractor.

Both stages use:
- Adam optimizer, lr=1e-3, weight_decay=1e-4
- `BCEWithLogitsLoss` with per-sample class weights (balanced)
- Early stopping on validation accuracy (patience=20)

## Data

Splits live in `data/data_splits/` and are generated by `scripts/split_data_reproducible.py`.
Each split file is organoid-level JSON (72% train / 8% val / 20% test, split by organoid to prevent leakage across timepoints):

```
data/data_splits/
  both_train_base.json
  both_val_base.json
  both_test_base.json
```

Labels come from Dy30 survey majority vote and are propagated to all earlier timepoints of the same organoid.

## Usage

Run from the repo root with the conda environment activated:

```bash
conda activate /net/projects2/promega

# Images + metabolites, concat fusion (symmetric 128-d projection)
python analysis/multimodal/train_multimodal.py \
    --backbone resnet \
    --use-metabolites \
    --fusion-strategy concat \
    --output-dir analysis/multimodal/outputs_multimodal

# With growth (delta) features — 8 or 10 metabolite dims instead of 4 or 5
python analysis/multimodal/train_multimodal.py \
    --backbone resnet \
    --use-metabolites \
    --use-growth-features \
    --fusion-strategy concat

# Gated fusion
python analysis/multimodal/train_multimodal.py \
    --backbone resnet \
    --use-metabolites \
    --fusion-strategy gated

# Smaller shared projection (give metabolites relatively more weight)
python analysis/multimodal/train_multimodal.py \
    --backbone resnet \
    --use-metabolites \
    --proj-dim 64

# Images only (ablation)
python analysis/multimodal/train_multimodal.py \
    --backbone resnet \
    --images-only
```

For a full hyperparameter sweep across backbones, fusion strategies, and input modes, submit the SLURM job array:

```bash
sbatch analysis/multimodal/run_multimodal.s
```

## Key Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--backbone` | `vit` | Image backbone: `resnet`, `efficientnet`, `vit` |
| `--fusion-strategy` | `concat` | `concat` or `gated` |
| `--proj-dim` | `128` | Shared projection dimension for both branches |
| `--use-metabolites` | off | Enable metabolite branch |
| `--use-growth-features` | off | Append delta features to metabolite vector |
| `--images-only` | — | Disable metabolite branch |
| `--metabolites-only` | — | Disable image branch |
| `--use-augmentation` | off | Random flips during training |
| `--days` | all | Restrict to specific days, e.g. `--days Dy03 Dy06` |

## Outputs

Per-day results are saved under `--output-dir/<day>/`:

| File | Contents |
|------|----------|
| `model.pth` | Best model weights (by val accuracy) |
| `metrics_test.json` | Accuracy, F1, recall, precision, specificity, AUC, confusion matrix |
| `organoid_predictions.csv` | Per-organoid predicted probability and TP/FP/TN/FN label |
| `training_curves.png` | Loss and accuracy curves |

A `results_summary.csv` and `metrics_by_day.png` are written at the top of `--output-dir` summarizing all days. An `overall/master_results.csv` accumulates results across multiple runs for easy comparison.
