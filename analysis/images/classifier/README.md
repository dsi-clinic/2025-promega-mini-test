# Image Classifier Analysis

This directory houses the data prep, training, evaluation, and reporting utilities for the organoid acceptance image classifier experiments.

## Layout Overview
- `data/` – preprocessed JSON datasets organized by resolution (`512x384`, `256x192`), vote strategy (`majority`, `complete`), and culture day.
- `outputs_*` – model checkpoints, metrics, and inference artifacts for each training sweep (augment vs. no augment, mask variants, soft labels, AUROC logging, etc.).
- `misclassified_images/`, `misclassifiedimages/` – CSV exports and stitched panels used during error analysis.
- `analysis/`, `crop_zoom_test/`, `stitched/` – notebooks, sanity-check figures, and stitched composites.
- `logs/` – SLURM logs generated by the `run_*.s` submission scripts.

## Core Scripts
- `generate_training_metadata.py` builds the per-day JSON payloads with propagated labels, optional raw vote dumps, and day normalization (Dy20/Dy21 → 20.5). Outputs populate `data/preprocessed/...`.
- `train_model_accuracy.py` is the primary trainer supporting ViT/ResNet/EfficientNet/custom CNN backbones, optional mask channels, augmentation flags, class weighting, AUROC tracking, and early unfreezing.
- `train_model_enhanced.py`, `train_model_soft_labels.py`, `train_model_deep_ensemble.py`, and `train_model_confusion_matrix.py` provide variants for alternative losses, ensembles, and richer diagnostics.
- `find_misclassified_images.py` plus `aggegate_mis.py` surface misclassified samples, merge per-model CSVs, and build intersection/union tables using auto-discovered ground-truth JSONs.
- `per_model_summary.py`, `plot_model_metrics.py`, and `plot_auc_curve.py` summarize accuracy/F1 by day and render plots for downstream reporting.
- `stitch_images_from_csv.py` creates contact sheets from CSV selections (e.g., misclassified or soft-label extremes) to speed qualitative review.

## Training Script Comparison
| Script | Primary goal | Target labels | Standout features | Default outputs |
| --- | --- | --- | --- | --- |
| `train_model_accuracy.py` | Baseline per-day classifier | Hard (`Accepted` / `Not Accepted`) | Optional mask branch, standard flip/jitter augmentation, AUROC tracking | `outputs_512x384_Regular_image_*` |
| `train_model_enhanced.py` | Stronger single-model baseline | Hard | Two-tier augmentation (geometric vs geometric+photometric), deterministic flip duplication, configurable normalization | `outputs_512x384_two_level_aug` |
| `train_model_soft_labels.py` | Uncertainty-aware training | Soft vote fractions (0–1) | BCE on soft targets, vote-count weighting, Brier/ROC metrics, automatic threshold sweep | `outputs_512x384_softlabels*` |
| `train_model_deep_ensemble.py` | Backbone sweep & per-day selection | Hard | Trains ViT/ResNet/EfficientNet per day without augmentation, early-stops each, picks best backbone by validation accuracy, aggregates per-day test metrics | `outputs_512x384_Regular_image_without_train_augment` |
| `train_model_confusion_matrix.py` | Diagnostic rerun on saved checkpoints | Hard | Reloads trained weights, recomputes split metrics, writes confusion matrices and ROC curves | Mirrors source run under `outputs_*` |

## Batch Execution
SLURM helpers (`run_*.s`) live alongside the Python scripts. `run_all.s` enqueues the four main sweeps:
1. `run_nomask_image.s` – base RGB images.
2. `run_nomask_overlay.s` – fluorescence overlay only.
3. `run_mask_image.s` – RGB + mask branch.
4. `run_mask_overlay.s` – overlay + mask branch.
Other runners (`run_accuracy.s`, `run_misclassified.s`, `run_soft_label.s`, etc.) launch targeted evaluations and assume execution from this directory so relative paths resolve.

## Typical Workflow
1. **Prepare data** – From repo root, run `python analysis/images/classifier/generate_training_metadata.py --all` (optionally pass `--classification` or `--emit-raw-votes`).
2. **Train models** – Submit one of the SLURM scripts or call `train_model_accuracy.py` directly with flags like `--backbones vit resnet` or `--use-mask`. Outputs land in the chosen `outputs_*` tree.
3. **Evaluate & visualize** – Use the summary scripts to create CSV tables, accuracy-by-day charts, ROC/AUC plots, and stitched misclassification panels. Aggregation tools help compare overlapping error cases across backbones.

## Notes
- Most scripts expect invocation from the repository root so relative paths stay valid.
- Dependencies include PyTorch, torchvision/timm, NumPy, pandas, matplotlib, seaborn, and scikit-learn; GPU acceleration is leveraged automatically when available.
- Outputs can be large; periodically prune stale `outputs_*` directories and intermediate exports.
