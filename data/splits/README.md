# Splits

Train/val/test assignments at the organoid level (per AGENTS.md rule #2: the same organoid must stay in one partition across all days to prevent data leakage). Each CSV has columns `organoid_id, split` and is loaded at runtime via `pipeline.splits.Splits`.

## Files

| File | Organoids | train / val / test | Provenance |
|---|---|---|---|
| `canonical_2026_winter.csv` | 220 | 158 / 18 / 44 | Repo default. Generated by `analysis.paper_2026_04.generate_splits` (seed=42) against the `all_data.json` produced in early April 2026. All paper-replication scripts default to this split via `Splits.canonical()`. |
| `harriet_2026_05.csv` | 220 | 158 / 18 / 44 | Pulled from PR #110 (`harriet/cleanup-data-splits`, May 2026). Same head counts as canonical and same 220 organoid IDs, but **differs in per-organoid partition assignment for 45 organoids (~20%)**. Provenance details unknown — generated by a different invocation of the split logic. Kept here as a labeled alternate, not as a replacement. |

## Usage

```python
from pipeline.splits import Splits
from pipeline.data_loader import OrganoidDataset, filters_for_mode

# Default — canonical split
ds = OrganoidDataset("data/all_data.json",
                     splits=Splits.canonical(),
                     filters=filters_for_mode("base"))

# Alternate
harriet = Splits.from_csv("data/splits/harriet_2026_05.csv", name="harriet_2026_05")
ds = OrganoidDataset("data/all_data.json", splits=harriet, filters=filters_for_mode("base"))

# Compare
print(Splits.canonical().agreement_with(harriet))
# {'shared_ids': 220, 'only_in_self': 0, 'only_in_other': 0,
#  'identical': 175, 'disagreements': 45,
#  'confusion': {('train','train'): 138, ('train','test'): 15, ('train','val'): 5, ...}}

# Ad-hoc random split (e.g., 80/20 from the filtered cohort)
splits = Splits.stratified_random(
    ds.organoid_labels(),
    ratios={"train": 0.8, "test": 0.2},
    seed=42,
    name="rand_80_20",
)
ds.apply_splits(splits)
```

## Why two splits coexist

The canonical CSV is what every published paper-replication number was computed against; switching the default partition would invalidate those results. Harriet's variant ships alongside as a labeled alternative for anyone who needs to compare or experiment with the alternate assignment.

If you regenerate splits (e.g., `make run ARGS="-m analysis.paper_2026_04.generate_splits"` after an `all_data.json` rebuild), write the output to a new name under this directory rather than overwriting `canonical_2026_winter.csv` — that file is treated as a frozen reference.

## Adding a new split

1. Drop the CSV into this directory with a descriptive name (e.g., `cohort_v2_seed7.csv`). Two columns required: `organoid_id`, `split`. Extra columns ignored.
2. Reference it from code via `Splits.from_csv("data/splits/cohort_v2_seed7.csv", name="cohort_v2_seed7")`.
3. Add a row to the Files table above with provenance.
